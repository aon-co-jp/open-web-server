# Гибридная сетевая архитектура (Резюме)


**Миссия (объединено в v0.2):** Гарантированная доставка и гарантированное чтение/запись данных, которые никогда не должны быть потеряны — платные предметы в 3D онлайн-играх, онлайн-финансы, онлайн-брокерские/ценные бумаги. Скорость и 4-уровневый транспорт существуют, чтобы служить этой миссии, а не конкурировать с ней.

**Цель:** Объединить 4-уровневый транспортный стек (сырой UDP → QUIC/HTTP3 → откат на TCP → мультиплексирование GraphQL-федерации) с гарантиями ACID `aruaru-db` и целостностью в стиле ZFS `open-raid-z`, охватывая `open-runo`, `poem-cosmo-tauri`, `open-web-server`, `aruaru-db` и `open-raid-z`.

**Текущее состояние:** Интеграция Poem в `aruaru-db` подтверждённо быстрая; совместимость SQL UPSERT с `open-runo` всё ещё не решена. В `open-raid-z` работают невыровненный ввод-вывод и инструмент миграции, но нативные типы Windows недоступны в Linux CI. `open-web-server` пока не проверен.

**Следующие шаги:** (1) исправить парсер UPSERT, (2) проверить `open-web-server`, (3) определить общий контракт согласования транспорта, (4) подключить контрольные суммы в стиле ZFS к пути записи БД, (5) быстрый путь QUIC/UDP строить в последнюю очередь.

Полные правила см. в `docs/HYBRID_NETWORK_ARCHITECTURE.md`. Примечание: документ составлен без живого веб-поиска; утверждения о «передовых технологиях» следует считать непроверенными до бенчмарков.
