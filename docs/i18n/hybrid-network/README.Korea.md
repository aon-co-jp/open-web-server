# 하이브리드 네트워크 아키텍처 (요약)


**사명 (v0.2 통합):** 절대 손실되어서는 안 되는 데이터에 대한 보장된 전달과 보장된 읽기/쓰기 — 3D 온라인 게임 유료 아이템, 온라인 금융, 온라인 증권 대상. 속도와 4계층 전송은 이 사명을 위해 존재하며 이와 경쟁해서는 안 된다.

**목표:** 4계층 전송 스택(순수 UDP → QUIC/HTTP3 → TCP 폴백 → GraphQL 페더레이션 멀티플렉싱)과 `aruaru-db`의 ACID 보장, `open-raid-z`의 ZFS 스타일 무결성을 `open-runo`, `poem-cosmo-tauri`, `open-web-server`, `aruaru-db`, `open-raid-z` 전반에 걸쳐 결합합니다.

**현재 상태:** `aruaru-db`의 Poem 통합은 고성능이 확인됨. `open-runo`와의 SQL UPSERT 호환성은 아직 미해결. `open-raid-z`는 비정렬 I/O와 마이그레이션 도구가 동작하지만, Linux CI에서는 Windows 전용 타입을 사용할 수 없음. `open-web-server`는 아직 감사되지 않음.

**다음 단계:** (1) UPSERT 파서 문제 해결, (2) `open-web-server` 감사, (3) 공유 전송 협상 계약 정의, (4) DB 쓰기 경로에 ZFS 스타일 체크섬 연결, (5) QUIC/UDP 고속 경로는 마지막에 구축.

자세한 내용은 `docs/HYBRID_NETWORK_ARCHITECTURE.md` 참조. 참고: 실시간 웹 조사 없이 작성되었으므로 "최첨단" 관련 주장은 벤치마크 전까지 미검증으로 취급할 것.

**조사 규칙:** 개발 및 유지보수 시 필요에 따라 Google 검색과 GitHub 조사를 적극적으로 수행할 것. 검색은 **일본어와 영어 모두**로 수행할 것 — 관련 정보(블로그 글, 권고문, 이슈)는 종종 한 언어에서만 나타나기 때문.

**업데이트 (v0.6):** 이번 세션에서 poem-cosmo-tauri는 이전에 보류되었던 여러 항목(gRPC 스트리밍/리플렉션, 비-Multipart 업로드, Redis를 통한 EDFS, 범위가 제한된 Cosmo Connect 필드)을 해결하고 오래된 문서 오류 2건을 수정했습니다. 자세한 내용과 환경 제약으로 실제로 보류 중인 항목은 본문 §0.6을 참조하세요.

**업데이트 (v0.7):** aruaru-db에 open-raid-z와 바이트 단위로 동일한 SHA-256 알고리즘을 사용하는 ZFS 호환 체크섬 계층을 구현하고 기존 ACID 트랜잭션과 하이브리드화했습니다. 모든 쓰기에 체크섬이 부여되고 모든 읽기에서 검증되며, zpool scrub에 해당하는 메서드로 손상된 행을 찾아냅니다. 자세한 내용과 다른 저장소로의 확장 단계는 §0.7을 참조하세요.

**다음 세션 지침 (v0.8):** 각 저장소 전반에 걸쳐 ZFS+ACID 하이브리드 융합을 계속하고, 일본어와 영어로 Google/GitHub 검색(기억에만 의존하지 않고)을 통해 조사한 후 4계층 4중 전송 스택(TCP-IP/UDP-IP/QUIC-MPQUIC/MPTCP-SCTP)을 구현할 것 — 게임 아이템/금융/증권/신용카드 데이터의 제로 손실을 위한 하나의 통합 파이프라인으로, 별개의 기능이 아니라. 자세한 내용은 §0.8 참조.

**업데이트 (v0.9):** "Poem과 브라우저 내 실행 기능을 포함한 Tauri 둘 다 처음부터 개발하여 완전 호환으로 재현한다"는 내용을 명시했습니다. open-runo/poem-cosmo-tauri README의 오래된 일방향 선행/미러 표현도 수정하고 문서 링크 목록을 왼쪽 정렬했습니다. 참고: 이번 세션에는 §0.8 조사 지침에 사용할 실시간 웹 검색 도구가 없었습니다.
