# بنية الشبكة الهجينة (ملخص)


**المهمة (مدمجة في الإصدار v0.2):** تسليم مضمون وقراءة/كتابة مضمونة للبيانات التي يجب ألا تُفقد أبدًا — عناصر مدفوعة في ألعاب ثلاثية الأبعاد عبر الإنترنت، والتمويل عبر الإنترنت، والأوراق المالية/الوساطة عبر الإنترنت. السرعة ومكدس النقل رباعي الطبقات موجودان لخدمة هذه المهمة، لا لمنافستها.

**الهدف:** الجمع بين مكدس نقل رباعي الطبقات (UDP خام ← QUIC/HTTP3 ← التراجع إلى TCP ← تعدد إرسال اتحاد GraphQL) مع ضمانات ACID الخاصة بـ `aruaru-db` وسلامة نمط ZFS الخاصة بـ `open-raid-z`، عبر `open-runo` و `poem-cosmo-tauri` و `open-web-server` و `aruaru-db` و `open-raid-z`.

**الوضع الحالي:** تم التحقق من أن تكامل Poem في `aruaru-db` سريع؛ لا تزال توافقية SQL UPSERT مع `open-runo` قضية مفتوحة. يحتوي `open-raid-z` على إدخال/إخراج غير محاذٍ وأداة ترحيل فعّالة، لكن أنواع Windows الأصلية غير متاحة على CI الخاص بلينكس. لم تتم مراجعة `open-web-server` بعد.

**الخطوات التالية:** (1) إصلاح محلل UPSERT، (2) مراجعة `open-web-server`، (3) تحديد عقد تفاوض نقل مشترك، (4) ربط المجاميع الاختبارية بنمط ZFS بمسار كتابة قاعدة البيانات، (5) بناء المسار السريع QUIC/UDP في النهاية.

راجع `docs/HYBRID_NETWORK_ARCHITECTURE.md` للقواعد الكاملة. ملاحظة: تم إعداد هذا المستند دون بحث ويب مباشر؛ يجب اعتبار ادعاءات "أحدث التقنيات" غير مؤكدة حتى إجراء القياس.

**قاعدة البحث:** يجب أن يقوم التطوير والصيانة بالبحث بنشاط على الويب (مثل Google) وعلى GitHub عند الحاجة — ويجب إجراء عمليات البحث **باللغتين اليابانية والإنجليزية معًا**، لأن المعلومات ذات الصلة (منشورات المدونات، التنبيهات، المشكلات) غالبًا ما تظهر بلغة واحدة فقط.

**تحديث (v0.6):** في هذه الجلسة، أغلق poem-cosmo-tauri عدة ثغرات كانت مؤجلة سابقًا (البث/الانعكاس عبر gRPC، الرفع بدون Multipart، EDFS عبر Redis، حقل Cosmo Connect محدود النطاق) وصحّح خطأين قديمين في التوثيق. راجع القسم §0.6 في المستند الكامل لسجل الإغلاق وما هو محظور فعليًا بسبب البيئة.

**تحديث (v0.7):** أصبح لدى aruaru-db الآن طبقة تجزئة متوافقة مع ZFS (خوارزمية SHA-256 مطابقة تمامًا لتلك الموجودة في open-raid-z)، مدمجة مع معاملات ACID الحالية — يتم حساب مجموع اختباري لكل عملية كتابة، ويتم التحقق من كل عملية قراءة، وتوجد طريقة مكافئة لـ zpool scrub تعثر على جميع الصفوف التالفة. راجع القسم §0.7 للتفاصيل.

**توجيه الجلسة القادمة (v0.8):** الاستمرار في دمج الهجين ZFS+ACID عبر جميع المستودعات، والبحث (عبر Google/GitHub باليابانية والإنجليزية معًا، وليس بالاعتماد على الذاكرة فقط) ثم تنفيذ مكدس النقل رباعي الطبقات ورباعي التكرار (TCP-IP/UDP-IP/QUIC-MPQUIC/MPTCP-SCTP) — كخط أنابيب واحد مدمج بلا فقدان للبيانات لعناصر الألعاب والتمويل والأوراق المالية وبيانات بطاقات الائتمان، وليس كميزات منفصلة. راجع القسم §0.8.
