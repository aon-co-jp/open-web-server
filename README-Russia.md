# open-web-server

## Веб-сервер на Rust + Poem, спроектированный так, чтобы платные предметы и финансовые данные никогда не терялись

open-web-server — критически важный веб-сервер, работающий 24/7/365, для таких
нагрузок, как покупка внутриигровых предметов в 3D-онлайн играх и платежи по
кредитным картам. Построен на **Rust + Poem** и работает вместе с aruaru-db и
open-runo через четырёхуровневую защитную архитектуру, благодаря чему сетевые сбои,
перезапуски процессов и повторные попытки никогда не приводят к двойному списанию
или незаметной потере данных.

## Пять опор

1. **Четырёхуровневая защищённая передача** (`open-web-server-wire`) — TLS 1.3 + взаимная аутентификация на основе HKDF + ChaCha20-Poly1305 + защита от повторного воспроизведения (seq/timestamp)
2. **Записи без потерь** (`open-web-server-ledger`) — предварительный WAL с обязательным `Idempotency-Key` + коммит в 3 шага
3. **Тесная интеграция с aruaru-db и open-runo** — `Клиент → open-web-server → open-runo → aruaru-db`
4. **Резервный путь UDP-IP** (`open-web-server-wire::udp_channel`, 2026-07-11) — параллельно с авторитетным TCP-коммитом отправляет зашифрованное и аутентифицированное (HMAC) UDP-уведомление по принципу best-effort, без повторной отправки (первая версия)
5. **Целевая архитектура: четырёхкратная избыточность транспорта и записи в БД** (пересмотрено 2026-07-11) — конечная цель: транспорт через TCP-IP + UDP-IP + QUIC/MPQUIC + MPTCP/SCTP, запись в PostgreSQL (свойства транзакций ACID: атомарность, согласованность, изолированность, долговечность) + aruaru-db + синхронную мультирегиональную репликацию + независимый журнал аудита. На данный момент реализованы TCP-IP, UDP-IP, а также ③QUIC (`quinn`, проверено реальным TLS1.3-рукопожатием) и WAL на PostgreSQL (`sqlx`, с реальными BEGIN/COMMIT; не проверено на реальном PostgreSQL из-за отсутствия доступа в этой песочнице); ④MPTCP/SCTP: подтверждено, что реализация/проверка на уровне ядра невозможна в этой Windows-песочнице; реализована пользовательская замена (`aggligator`, `open-web-server-wire::mptcp_channel`, явно не настоящий ядерный MPTCP/SCTP), проверенная реальным TCP round-trip по loopback (2026-07-13). Остальные пути резервирования БД ещё не начаты (подробности: [README-Japan.md](README-Japan.md#6-目標アーキテクチャ-通信層dbの四重化), [CLAUDE.md](CLAUDE.md)). **Запланированная новая разработка**: связать коммиты aruaru-db со снапшотами ZFS (open-raid-z) — устоявшейся техники не найдено, но это расценивается как новая, реализуемая находка, запланированная на будущий проход.

## Быстрый старт

```bash
cargo run -p aruaru-server -- --data ./data --raft-id 1
cargo run -p open-runo-gateway
OPEN_RUNO_ENDPOINT=https://127.0.0.1:8443 cargo run -p open-web-server-gateway
```

## Структура (4 крейта)

`open-web-server-core` (доменные модели/ошибки), `open-web-server-wire` (четырёхуровневая защищённая передача),
`open-web-server-ledger` (идемпотентный WAL + коммит в 3 шага), `open-web-server-gateway` (шлюз на Poem).
